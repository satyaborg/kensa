<Check>Start with [Skills](/skills) if you want the most hands-off experience. Reach for [CLI Reference](/cli), [MCP Server](/mcp), or [CI Integration](/ci) when you need explicit automation surfaces.</Check>

<Columns cols={2}>
  <Card title="Skills" href="/skills" cta="Use the guided workflow" arrow="true">
    Best for coding-agent-first workflows where you want kensa to discover the right next step for you.
  </Card>
  <Card title="CLI Reference" href="/cli" cta="See every command" arrow="true">
    Best when you want explicit control over init, run, judge, report, analyze, and MCP serving.
  </Card>
  <Card title="MCP Server" href="/mcp" cta="Integrate with MCP clients" arrow="true">
    Best when you want Cursor, Codex, Claude Desktop, or other MCP-aware clients to call kensa as tools.
  </Card>
  <Card title="CI Integration" href="/ci" cta="Gate changes in CI" arrow="true">
    Best when you want deterministic checks and optional LLM judging on pushes and pull requests.
  </Card>
</Columns>

## Which path fits your team?

| Situation | Best fit |
|---|---|
| "I want my coding agent to handle most of this for me" | [Skills](/skills) |
| "I want exact commands and flags" | [CLI Reference](/cli) |
| "I want other tools or clients to drive evals programmatically" | [MCP Server](/mcp) |
| "I want evals to block or comment on pull requests" | [CI Integration](/ci) |
